---

Desentralisasi dilakukan secara bertahap karena kualitas tata kelola lebih penting daripada kecepatan tata kelola.

**Fase 1 (bulan 0-6 pasca-TGE).** Multisig Foundation mengendalikan pembaruan. Kemampuan jeda darurat tersedia.

**Fase 2 (bulan 6-18 pasca-TGE).** Pemungutan suara pemegang token diaktifkan untuk parameter yang tidak bersifat kritis. Komunitas dapat mengajukan proposal tata kelola.

**Fase 3 (bulan ke-18+ pasca-TGE).** Kontrol utama DAO atas semua parameter protokol. Hak veto Foundation berakhir, dengan kemampuan darurat admin yang terbatas tetap dipertahankan demi keamanan.
