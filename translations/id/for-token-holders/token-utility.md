**Kepemilikan.** $P2P adalah token kepemilikan. Dana treasury dan otoritas pencetakan dikontrol oleh pemegang token melalui futarchy MetaDAO di Solana, dan kekayaan intelektual protokol, parameter, serta peningkatan dikontrol oleh pemegang token melalui Governor on-chain di Base, bukan oleh tim, yayasan, atau entitas tunggal mana pun. Ini berarti token membawa hak tata kelola yang dapat ditegakkan, termasuk mekanisme untuk mengalihkan kendali sebagaimana dijelaskan dalam "Mengapa Token Ini Ada." Keputusan yang memengaruhi pasokan token (pencetakan) harus melalui pasar keputusan futarchy MetaDAO, di mana peserta mempertaruhkan modal pada apakah suatu proposal meningkatkan atau menurunkan nilai, dan prediksi pasar yang menentukan hasilnya.

**Tata Kelola.** Pemegang token memberikan suara tentang parameter protokol seperti biaya, batas, aturan merchant, dan konfigurasi oracle melalui Governor on-chain, di mana satu $P2P yang di-stake = satu suara, dengan delegasi. Alokasi treasury diatur secara terpisah melalui futarchy MetaDAO di Solana.

**Staking.** Circle Admin men-stake $P2P untuk mengoperasikan jaringan merchant. Anggota komunitas mendelegasikan USDC ke dalam pool Circle untuk mendapatkan bagian dari reward merchant. Merchant men-stake USDC sebagai modal kerja. Desain staking menyelaraskan insentif dengan mensyaratkan komitmen ekonomi di setiap lapisan.

**Distribusi biaya.** Pendapatan protokol disalurkan kepada para peserta.

| Penerima               | Bagian Pendapatan                                                             |
| ---------------------- | ----------------------------------------------------------------------------- |
| Merchant + Delegator   | 53,33%                                                                        |
| Treasury               | 20%, peta jalan menuju 33% bagian beli-dan-bakar (diatur melalui futarchy MetaDAO) |
| Pool Asuransi          | 17,78%                                                                        |
| Circle Admin           | 8,89%                                                                         |

Tidak ada pihak tunggal yang menguasai mayoritas pendapatan protokol. Merchant memperoleh bagian terbesar karena mereka menyediakan modal kerja dan tenaga operasional. Kontribusi treasury mengaitkan penggunaan protokol dengan treasury jaringan. Tata kelola dapat mengarahkan dana ini menuju beli-dan-bakar atau kegunaan lain melalui pasar keputusan futarchy MetaDAO. Pool asuransi ada agar sengketa tidak menjadi biaya yang dieksternalisasi.
